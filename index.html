Docker:

MySQL:
MySQL was our DB of choice to manage the backend data for our project. It had two tables: A user table that contained the information like their id, email, a current balance; and a table of transactions that were paired with the user table. The transaction table contained what type of transaction was made, if stocks were involved, what user id it is paired with, etc. I created the entity class for the User, and queried through the transaction table when creating a portfolio history for a user.
